Mars Genesis: Multi-Agent AI Simulation

Mars Genesis is a live multi-agent AI simulation that demonstrates emergent behavior in autonomous AI agents. Two AI commanders with distinct personality profiles lead the same colony of 100 colonists through dynamically generated crises. Their HEXACO personality traits shape every decision: which risks to take, which departments to trust, and how to allocate scarce resources. The result is two genuinely different civilizations emerging from identical starting conditions.

This is not a scripted demo. Starting from Turn 2, an AI Crisis Director generates unique crises for each timeline based on colony state, decision history, and resource levels. The same seed guarantees the same colonist roster and RNG stream. Only leadership personality differs. Every divergence is traceable.

Architecture: What Controls What

The simulation has three layers, and understanding which layer controls what is key to understanding the results.

  • Deterministic kernel (no LLM): births, deaths, aging, bone density loss, radiation accumulation, resource production, and career progression. All driven by seeded RNG (Mulberry32). The kernel decides who dies, who is born, and how resources change between turns. Same seed always produces the same outcomes. The LLM has no say in these mechanics.
  • AI agents (LLM-driven): crisis generation, department analysis, tool forging, and commander decisions. The commander and 5 department heads are LLM agents. They interpret the crisis, forge analytical tools, recommend policies, and decide strategy. Their decisions feed into the kernel as bounded numerical effects.
  • LLM-as-judge (tool review only): reviews forged tool code for safety, correctness, determinism, and bounded execution. The judge does NOT determine simulation outcomes, deaths, or events. It only decides whether a forged tool's code is safe to run.

How Deaths and Births Work

Deaths and births are entirely deterministic. The LLM does not decide who lives or dies.

  • Natural mortality: age-based probability per year. 1% at age 60, 3% at 70, 8% at 80, 20% at 90. Seeded RNG rolls determine each colonist's fate.
  • Radiation deaths: colonists accumulating over 1000 mSv get +2% annual mortality. Over 2000 mSv adds another +5%. Mars surface radiation is 0.67 mSv/day (244 mSv/year) per Hassler et al. 2014 Curiosity RAD measurements.
  • Births: adults age 20-42 with colony morale above 40% and food reserves above 6 months have an 8% birth probability per year-delta. Maximum 3 children per parent. All Mars-born children start with 88% bone density based on Sibonga et al. 2019 ISS bone density data scaled to Mars gravity (0.38g).
  • Resource effects: bone density degrades 0.3-0.5% per year on Mars per Hughson et al. 2018. Radiation accumulates per Cucinotta et al. 2010 dose-response model. These are kernel mechanics, not LLM decisions.

How Tool Forging Works

Tool forging is the primary emergent capability demonstration. Here is exactly what happens:

  1. Agent identifies need: A department agent (Medical, Engineering, etc.) analyzes the crisis and decides it needs a computational model that doesn't exist yet. For example, the Medical agent facing a radiation crisis decides it needs a "cumulative dose risk calculator."
  2. Agent writes code: The agent specifies the tool's name, description, input schema, output schema, implementation code (a sandboxed JavaScript function), and test cases.
  3. Judge reviews: An LLM-as-judge evaluates the tool across 5 dimensions: safety (no data exfiltration or resource exhaustion), correctness (outputs match test cases), determinism (consistent results), bounded execution (completes within time/memory limits), and input validation. The judge scores each dimension 0-1.
  4. Tool executes: If approved (all scores above threshold), the tool runs in an isolated V8 sandbox with the current colony data as input. It produces numerical output (risk scores, projections, calculations).
  5. Output informs decision: The tool's computed output is included in the department's report to the commander, who uses it alongside other department analyses to make a strategic decision.

Tools do NOT directly change colony state. They produce analysis that informs decisions. The commander's selected policy effects change state through the kernel.

Are Colonists Individual Agents?

Yes. All alive colonists generate individual reactions after each crisis outcome using gpt-4o-mini. Each colonist receives their HEXACO personality profile, health stats, social ties, and the crisis context, then produces a 1-2 sentence reaction with mood and intensity. All 100 calls run in parallel (~10 seconds per turn, ~$0.14 for a full 12-turn simulation).

Colonists are NOT micro-agents with persistent sessions. Their health, careers, births, and deaths are deterministic kernel mechanics. But their reactions are real LLM output shaped by their individual personality and circumstances. A high-openness Mars-born teenager reacts differently to a governance crisis than a high-conscientiousness Earth-born engineer.

The simulation has 107+ LLM agents per turn: 1 commander, 5 department heads, 1 Crisis Director, and ~100 colonist reaction agents. Department heads and the commander use the full-capability model (gpt-5.4 or Claude). Colonists use the cheap model (gpt-4o-mini) for cost efficiency.

How Outcomes Are Determined

When the commander selects an option (risky or conservative), the outcome is determined by:

  1. The crisis defines a base success probability for the risky option (e.g., 55%)
  2. Colony morale above 70% adds +10% to success chance
  3. Food reserves above 12 months add +5%
  4. Population above 150 subtracts -5% (overcrowding penalty)
  5. A seeded RNG roll determines success or failure

The LLM does not determine whether the outcome succeeds or fails. The seeded RNG does. This means the same decision in the same colony state with the same seed always produces the same result.

What the Numbers Mean

  • Population: alive colonists. Kernel-driven births/deaths.
  • Morale: colony wellbeing 0-100%. Affected by food, overcrowding, policy effects.
  • Food: months of reserves. Below 6 triggers escalation from the Crisis Director.
  • Deaths: cumulative. Age + radiation mortality from the kernel.
  • Tools Forged: computational tools the AI agents invented and the judge approved.
  • Citations: DOI-linked research papers used by department agents in their analysis.
  • HEXACO: O(penness) C(onscientiousness) E(xtraversion) A(greeableness) Em(otionality) H(onesty-Humility).
  • Personality Drift: HEXACO changes from leadership influence, role activation, and outcome feedback.

FAQ

  • Is this scripted? Turn 1 (Landfall) and the final turn (Legacy Assessment) are fixed milestones. Every other turn uses the AI Crisis Director, which generates crises based on colony state. Different leader personalities create different colony states which trigger different crises.
  • Do tools define events? No. The Crisis Director generates events based on colony state. Tools are analytical instruments that department agents create to better understand the crisis. A radiation dose calculator doesn't cause a radiation event. It helps the Medical department quantify the risk so the commander can decide.
  • Is the judge impartial? The judge reviews forged tool code, not simulation outcomes. It scores safety, correctness, determinism, and bounded execution. It uses the same LLM model as the commander (configurable, default gpt-5.4). It is a code reviewer, not a simulation referee.
  • Are colonists agents? Yes. All ~100 alive colonists generate individual reactions each turn using gpt-4o-mini in parallel (~10 seconds, ~$0.006 per turn). Each colonist's HEXACO personality, health, and social context shapes their reaction. Department heads and commander use full-capability models.
  • What determines who dies? The deterministic kernel. Age-based mortality probability + radiation cumulative dose (Cucinotta 2010), rolled against seeded RNG. The LLM has no influence on death mechanics.
  • What is HEXACO? A six-factor personality model from psychology research. Unlike Big Five, it includes Honesty-Humility. Each trait 0-1 shapes how the commander evaluates risk, trusts departments, and allocates resources. Personality drift grounded in trait activation theory (Tett & Burnett 2003).
  • Can I use my own leaders? Yes. Settings tab: leader names, archetypes, HEXACO sliders, instructions, resources, custom events. Export/import via JSON. Share configs via URL.
  • What models does it use? Configurable per role: commander (gpt-5.4/Claude Opus), departments (gpt-5.4-mini/Claude Sonnet), judge (same as commander). All in Settings.
  • Can I save and replay? Save Game downloads all events as JSON. Load Game replays them. Reports tab shows full side-by-side. Data persists across page refresh via localStorage.
  • How do citations work? The knowledge base contains 25+ DOI-linked research papers on Mars science (radiation: Hassler 2014, bone density: Sibonga 2019, psychology: Basner 2014, terraforming: Jakosky 2018). When live search is enabled, department agents also search the web via AgentOS WebSearchService using Serper, Tavily, Firecrawl, and Brave in parallel with Cohere rerank-v3.5 neural reranking. Citations appear as clickable hyperlinks in department cards.
  • What research APIs are supported? All optional. Serper (Google search), Tavily (AI-optimized search), Firecrawl (web scraping + markdown extraction), Brave (privacy-focused search), and Cohere (neural reranking). More providers = richer citations. Configure in Settings tab or .env file.

Built with AgentOS

Mars Genesis runs on AgentOS, an open-source TypeScript framework for building autonomous AI agents with personality, memory, tool use, and multi-agent coordination. AgentOS provides the agent() API, EmergentCapabilityEngine for runtime tool forging, EmergentJudge for LLM-as-judge tool review, and ForgeToolMetaTool that agents call to create new tools.
