Mars Genesis: Multi-Agent AI Simulation

Mars Genesis is a live multi-agent AI simulation that demonstrates emergent behavior in autonomous AI agents. Two AI commanders with distinct personality profiles lead the same colony of 100 colonists through dynamically generated crises spanning 50 years on Mars. Their HEXACO personality traits shape every decision: which risks to take, which departments to trust, and how to allocate scarce resources. The result is two genuinely different civilizations emerging from identical starting conditions.

This is not a scripted demo. Starting from Turn 2, an AI Crisis Director generates unique crises for each timeline based on colony state, decision history, and resource levels. The same seed guarantees the same colonist roster and RNG stream. Only leadership personality differs. Every divergence is traceable and explainable.

How the Simulation Works

  • Deterministic simulation kernel owns canonical state: colonist generation, aging, births, deaths, career progression, bone density, radiation exposure, and resource accounting. All driven by seeded RNG (Mulberry32) for full reproducibility.
  • Five department agents (Medical, Engineering, Agriculture, Psychology, Governance) analyze each crisis independently. Each agent has its own HEXACO personality that drifts over time based on leadership influence and crisis outcomes.
  • Runtime tool forging lets department agents invent computational tools on the fly: radiation dose calculators, food security projectors, structural load analyzers. Each tool is reviewed by an LLM-as-judge for safety, correctness, and determinism before activation. Nobody pre-programmed these tools. The agent decides what it needs and builds it.
  • Personality evolution changes promoted colonists' six-factor HEXACO traits through three forces grounded in real psychology research: leader pull (trait convergence toward the commander), role pull (department activates specific traits per Tett & Burnett 2003), and outcome pull (successful risks boost openness, failures boost conscientiousness).
  • Emergent Crisis Director generates crises tailored to each timeline's colony state. Struggling colonies face survivable crises. Thriving colonies face existential ones. Crises reference consequences of prior decisions and never repeat the same category twice in a row.
  • Typed policy effects make commander decisions materially change kernel state through bounded numerical effects (resource shifts, capacity expansions, governance changes), not just narrative prose.

What the Numbers Mean

  • Population: alive colonists. Starts at 100. Changes from births (deterministic from age/relationship model), deaths (aging, radiation, crisis), and immigration decisions.
  • Morale: colony wellbeing from 0-100%. Affected by food reserves, overcrowding relative to life support capacity, and commander policy decisions.
  • Food: months of food reserves remaining. Below 6 months triggers resource crisis escalation from the director.
  • Deaths: cumulative colonist deaths. Tracked from kernel progression events (aging, radiation exposure, crisis outcomes).
  • Tools Forged: computational tools invented by department agents at runtime. Each tool is sandboxed, judge-reviewed, and produces real computed output that influences decisions.
  • HEXACO traits: O=Openness (curiosity, experimentation), C=Conscientiousness (discipline, thoroughness), E=Extraversion (sociability, leadership visibility), A=Agreeableness (cooperation, trust), Em=Emotionality (anxiety, empathy), HH=Honesty-Humility (sincerity, fairness).
  • Personality Drift: how colonist traits shifted this turn. Shows the numerical HEXACO changes from leadership influence, role activation, and crisis outcome feedback.

FAQ

  • Is this scripted? Turn 1 (Landfall) and the final turn (Legacy Assessment) are fixed milestones. Every other turn uses an AI Crisis Director that generates crises based on what actually happened in each timeline. Two runs with different leader personalities produce different crises, different tools, and different civilizations.
  • What is HEXACO? A six-factor personality model from psychology research. Unlike Big Five, HEXACO includes Honesty-Humility as a distinct factor. Each trait ranges from 0 (low) to 1 (high) and shapes how the commander evaluates risk, trusts departments, and allocates resources.
  • What is tool forging? Department agents can request the creation of new computational tools at runtime. The agent describes what it needs (a radiation dose calculator, a food security projector), writes the code in a sandboxed environment, and an LLM judge reviews it for safety and correctness. Approved tools produce real numerical outputs that inform the commander's decision.
  • What is the deterministic kernel? The simulation engine that handles births, deaths, aging, and resource accounting using seeded pseudo-random number generation. Same seed always produces the same colonist roster and the same progression events. Only the AI-driven decisions differ between runs.
  • Can I use my own leaders? Yes. The Settings tab lets you configure leader names, archetypes, HEXACO personality sliders, commander instructions, colony resources, and custom events. Export and share configs via URL or JSON.
  • What models does it use? Configurable per role. Commanders default to gpt-5.4 or Claude Opus. Departments default to gpt-5.4-mini or Claude Sonnet/Haiku. Tool forge judges use the commander model. All configurable in Settings.
  • Can I save and replay simulations? Yes. The Save Game button downloads all SSE events as JSON. Load Game replays them visually. The Reports tab generates a full side-by-side comparison.

Built with AgentOS

Mars Genesis runs on AgentOS, an open-source TypeScript framework for building autonomous AI agents with personality, memory, tool use, and multi-agent coordination. AgentOS provides the agent() API, EmergentCapabilityEngine for runtime tool forging, EmergentJudge for LLM-as-judge tool review, and ForgeToolMetaTool that agents call to create new tools.
