1. Install
npm install paracosm
2. Define your world as JSON
// my-world.json — defines the world, not the leaders // Leaders are configured separately so the same world // can be run with different personality matchups { "id": "submarine-habitat", "labels": { "name": "Deep Ocean Habitat", "populationNoun": "crew", "settlementNoun": "habitat", "currency": "credits" }, "setup": { "defaultTurns": 8, "defaultPopulation": 50, "defaultStartYear": 2040, "defaultSeed": 42 }, "departments": [ { "id": "life-support", "label": "Life Support", "role": "Chief Life Support Officer", "instructions": "Analyze O2 levels, CO2 scrubbing capacity, water recycling." }, { "id": "engineering", "label": "Engineering", "role": "Chief Engineer", "instructions": "Analyze hull integrity, pressure systems, power generation." } ], "metrics": [ { "id": "population", "format": "number" }, { "id": "morale", "format": "percent" } ] }
3. Compile the world, define two leaders, run them in parallel
import { compileScenario } from 'paracosm/compiler'; import { runSimulation } from 'paracosm/runtime'; import worldJson from './my-world.json'; // 1. Compile JSON into a runnable ScenarioPackage (~$0.10, cached to disk) const scenario = await compileScenario(worldJson, { provider: 'anthropic', model: 'claude-sonnet-4-6', }); // 2. Define two commanders with different HEXACO personalities // Same world, same crises — different decisions const leaderA = { name: 'Captain Reyes', archetype: 'The Pragmatist', colony: 'Station Alpha', hexaco: { openness: 0.4, conscientiousness: 0.9, extraversion: 0.3, agreeableness: 0.6, emotionality: 0.5, honestyHumility: 0.8 }, instructions: 'You lead by protocol. Safety margins first.', }; const leaderB = { name: 'Captain Okafor', archetype: 'The Innovator', colony: 'Station Beta', hexaco: { openness: 0.9, conscientiousness: 0.4, extraversion: 0.8, agreeableness: 0.5, emotionality: 0.3, honestyHumility: 0.6 }, instructions: 'You lead by experimentation. Push boundaries.', }; // 3. Run both in parallel — same seed, same crises, different outcomes const [resultA, resultB] = await Promise.all([ runSimulation(leaderA, [], { scenario, maxTurns: 8, seed: 42, onEvent(e) { console.log('A:', e.type, e.data?.title); } }), runSimulation(leaderB, [], { scenario, maxTurns: 8, seed: 42, onEvent(e) { console.log('B:', e.type, e.data?.title); } }), ]); // resultA.finalState vs resultB.finalState — the divergence

Core Capabilities

What Paracosm Does

Paracosm is a swarm simulation engine, not a chat interface. Agents analyze data, write code, cite papers, and forge computational models inside sandboxed isolates. The engine generates crises from swarm state, not from a script. Every run produces different outcomes from the same starting conditions.

Runtime Tool Forging

A medical officer invents a radiation dose calculator. An engineer writes a structural load analyzer. These tools don't exist in the codebase. The agent writes them, they execute in an isolated V8 sandbox, and an LLM judge reviews output before it enters the decision pipeline.

AgentOS ForgeToolMetaTool, 10s timeout, 128MB memory limit

HEXACO Personality Drift

Each agent carries six continuous personality traits (Openness, Conscientiousness, Extraversion, Agreeableness, Emotionality, Honesty-Humility). These values shift over time through three mechanisms: leader influence, role-specific pressures, and crisis outcome reinforcement. The same colonist under two different commanders becomes a measurably different person by turn 12.

Ashton & Lee, "Empirical, Theoretical, and Practical Advantages of the HEXACO Model," PSPR 11(2), 2007

Emergent Crisis Director

The Crisis Director reads swarm metrics, resource reserves, population health, prior decisions, and the tools agents have built. It generates a crisis that targets actual weak points in the simulation state. If a commander neglected agriculture in turn 3, the food crisis in turn 7 is a consequence, not a coincidence.

cf. Riedl & Bulitko, "Interactive Narrative: An Intelligent Systems Approach," AI Magazine 34(1), 2013

Deterministic Kernel

The simulation kernel uses Mulberry32, a 32-bit seeded PRNG. Given the same seed, the kernel produces identical agent rosters, lifecycle schedules, and promotion sequences. The only source of variation is the AI commander's decisions. Two runs with the same seed and different personalities produce different outcomes from provably identical starting conditions.

cf. Law & Kelton, Simulation Modeling and Analysis, 5th ed., McGraw-Hill, 2015

Scenario Compiler

Write a JSON file describing departments, metrics, and world rules. The compiler calls generateText() to produce TypeScript hooks for progression, crisis generation, milestones, and political dynamics. A Mars colony, a lunar outpost, an Antarctic station, a corporate org, a submarine crew: same compiler, different JSON, different swarm. Cost is roughly $0.10 per compile. Results cache to disk.

compileScenario() in paracosm/engine/compiler

Semantic Research Pipeline

Swarm agents search academic literature before making recommendations. The pipeline queries Serper, Tavily, Firecrawl, and Brave in parallel, then applies Cohere Rerank v3.5 for neural relevance scoring. Citations include DOI links. A medical agent recommending radiation shielding cites actual NASA dosimetry guidelines, not a hallucinated source.

Simulation Loop

Anatomy of a Turn

A turn takes 30 to 90 seconds. The engine runs four phases in sequence. Each phase feeds data forward. By the end of a turn, the swarm state has changed, agents have reacted, and personality traits have shifted.

01

Crisis Director

Reads swarm state, prior decisions, and tool intelligence. Generates a crisis that targets actual weaknesses.

02

Department Analysis

Five department agents analyze the crisis in parallel. Each cites research and can forge new computational tools.

03

Commander Decision

The AI commander weighs department input through the lens of their HEXACO personality profile.

04

Kernel Resolution

Deterministic state machine applies effects. Agents react, form opinions, update relationships. HEXACO traits drift.

Divergence

Same seed. Different commander. Different civilization.

The kernel guarantees identical starting conditions. Seed 950 produces the same 100 agents with the same profiles, the same resource levels, and the same initial crises. The only variable is the commander's personality. By turn 3, the swarms are measurably different.

Scenarios

Two ScenarioPackages ship with the engine

A ScenarioPackage is a JSON file plus runtime hooks. It defines departments, metrics, crisis templates, progression rules, and milestone conditions. The engine reads the package, spawns the agent swarm, and runs the simulation. No engine code changes required between scenarios.

Flagship

Mars Genesis

100-person Mars colony across 50 simulated years. Five departments (Medical, Engineering, Agriculture, Psychology, Governance) analyze each crisis. Agents track bone density, radiation exposure, food reserves, morale, and political faction alignment. The Crisis Director uses all of this to generate crises that target the colony's weakest points.

Architecture

Three layers, strict boundaries

The engine layer owns types and the deterministic kernel (71 source files). The runtime layer orchestrates simulations via 8 hook interfaces (18 files). The CLI layer serves the dashboard, batch runner, and HTTP server (45 files). No circular dependencies. The engine never imports from runtime.

FAQ

Frequently asked questions

What is Paracosm?
An open-source TypeScript swarm simulation engine available on npm (npm install paracosm). You define a world as a ScenarioPackage: departments, metrics, crisis templates, progression hooks. The engine spawns an autonomous agent swarm, generates crises from simulation state, and resolves outcomes through a seeded deterministic kernel. Two scenarios ship with the package. You can compile new ones from JSON.
How does tool forging work?
When a department agent needs a calculation the codebase doesn't provide, it writes one. A medical officer facing a radiation crisis writes a dose calculator. An engineer writes a structural load model. The code executes in an isolated V8 sandbox with memory and time limits. An LLM judge reviews the output for safety and correctness before it enters the decision pipeline. The tool persists for future turns. None of these tools exist in the source code. The agents invent them at runtime.
Implementation: AgentOS EmergentCapabilityEngine + EmergentJudge, sandboxed via isolated-vm
What is the HEXACO personality model?
HEXACO is a six-factor personality framework from personality psychology: Honesty-Humility, Emotionality, Extraversion, Agreeableness, Conscientiousness, and Openness to Experience. Each agent in Paracosm carries six continuous values between 0 and 1. These values shift across turns through three drift mechanisms: leader influence (agents' traits pull toward their commander's profile), role activation (a chief engineer's conscientiousness trends upward), and outcome reinforcement (surviving a crisis through bold action increases openness).
Ashton, M.C. & Lee, K. (2007). "Empirical, Theoretical, and Practical Advantages of the HEXACO Model of Personality Structure." Personality and Social Psychology Review, 11(2), 150-166. doi:10.1177/1088868306294907
Is the simulation deterministic?
The kernel is fully deterministic. It uses Mulberry32, a 32-bit seeded PRNG. Given seed 950, the kernel produces identical agent rosters, lifecycle schedules, promotion sequences, and resource starting values on every run. The divergence between two parallel swarms comes entirely from the AI commander's decisions. Fix the seed, vary the personality, measure the outcome.
PRNG methodology follows Law, A.M. & Kelton, W.D. Simulation Modeling and Analysis, 5th ed., McGraw-Hill, 2015
What LLM providers work with Paracosm?
OpenAI (GPT-5.4, GPT-5.4-mini) and Anthropic (Claude Sonnet 4.6, Claude Haiku 4.5). You can assign different models to different roles. The crisis director and commander might use GPT-5.4 while department agents and the tool judge use GPT-5.4-mini to manage costs. Configuration is per-role in the setup panel or via the API.
Can I build my own scenario?
Two paths. If you want full control, write a ScenarioPackage in TypeScript with custom hooks for progression, crisis generation, milestones, and politics. If you want zero-code authoring, write the scenario as JSON and run it through the Scenario Compiler. The compiler calls generateText() to produce all eight hook functions. Compilation costs about $0.10 in API calls and caches results to disk. An Antarctic station, a submarine crew, a corporate org, a generation ship: different JSON, same engine, different swarm.
What verticals does this apply to?
Defense and intelligence use it for wargaming and scenario planning. Corporate strategy teams model acquisition outcomes and leadership dynamics. Game studios generate procedural NPC civilizations and emergent narratives. Academic researchers run controlled experiments in multi-agent decision-making. Government agencies simulate policy impacts before implementation. The common thread: any domain where testing decisions before making them reduces risk.
Is Paracosm free?
The engine is open source under Apache-2.0. You can use it in commercial products without restriction. Install from npm, fork the repo, build on it. Hosted tiers for managed deployment, auth, persistence, and scenario authoring are on the roadmap. Contact team@frame.dev for early access.
