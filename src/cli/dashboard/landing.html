PARACOSM

A structured world model — for actions, not pixels.

While Sora, Genie 3, and Marble generate perceptual futures, Paracosm simulates actionable possibilities: a typed ScenarioPackage, deterministic seeded kernel, HEXACO-personality leaders, and byte-equal replay. Counterfactual world simulation in the lineage of Kirfel et al 2025.

Compile a prompt, brief, URL, or scenario JSON into a typed ScenarioPackage. Run HEXACO-personality leaders against a deterministic seeded kernel. Swap one variable and watch the trajectories diverge.

1. Install (npm, pnpm, or bun)
# Any of these work; paracosm ships ESM + subpath exports npm install paracosm pnpm add paracosm bun add paracosm
2. Define or ground your world
// my-world.json: the durable world contract // Leaders are configured separately so the same world // can be run with different personality matchups { "id": "submarine-habitat", "labels": { "name": "Deep Ocean Habitat", "populationNoun": "crew", "settlementNoun": "habitat", "currency": "credits" }, "setup": { "defaultTurns": 8, "defaultPopulation": 50, "defaultStartYear": 2040, "defaultSeed": 42 }, "departments": [ { "id": "life-support", "label": "Life Support", "role": "Chief Life Support Officer", "instructions": "Analyze O2 levels, CO2 scrubbing capacity, water recycling." }, { "id": "engineering", "label": "Engineering", "role": "Chief Engineer", "instructions": "Analyze hull integrity, pressure systems, power generation." } ], "metrics": [ { "id": "population", "format": "number" }, { "id": "morale", "format": "percent" } ] }

Positioning

Where paracosm sits in the world-model landscape

Paracosm is a structured world model (Xing 2025; ACM CSUR 2025) and a counterfactual world simulation model (Kirfel et al, 2025). The 2026 landscape has several adjacent categories. Paracosm fits exactly one. Naming the others up front saves everyone a misclassification cycle.

Structured / LLM-based / counterfactual world simulator. Prompt/document/URL-grounded, JSON-contract-backed state space, seeded deterministic kernel, LLM-driven events and specialist analyses, HEXACO-personality leaders, runtime tool forging in hardened node:vm sandboxes. One universal Zod-validated RunArtifact spans turn-loop civ sims, batch-trajectory digital twins, and batch-point forecasts.

Core Capabilities

What Paracosm Does

Paracosm is a structured world model, not a chat interface. Specialist agents analyze data, write code, cite papers, and forge computational models inside hardened node:vm sandboxes. The Event Director generates events from world state, not from a script. Two runs on the same seed diverge only because their leaders have different HEXACO personalities.

Runtime Tool Forging

A medical officer invents a radiation dose calculator. An engineer writes a structural load analyzer. These tools don't exist in the codebase. The agent writes them, they execute in a hardened node:vm sandbox, and an LLM judge reviews output before it enters the decision pipeline.

AgentOS ForgeToolMetaTool, 10s timeout, observed heap usage

HEXACO Personality Drift

Each agent carries six continuous personality traits (Openness, Conscientiousness, Extraversion, Agreeableness, Emotionality, Honesty-Humility). These values shift over time through three mechanisms: leader influence, role-specific pressures, and outcome reinforcement. The same agent under two different leaders becomes a measurably different entity by turn 12.

Ashton & Lee, "Empirical, Theoretical, and Practical Advantages of the HEXACO Model," PSPR 11(2), 2007

Emergent Event Director

The Event Director reads world state, resource levels, population health, prior decisions, and the tools agents have built. It generates events that target actual weak points in the simulation: crises, opportunities, disruptions, transitions. If a leader neglected agriculture in turn 3, the food shortage in turn 7 is a consequence, not a coincidence.

cf. Riedl & Bulitko, "Interactive Narrative: An Intelligent Systems Approach," AI Magazine 34(1), 2013

Deterministic Kernel

The simulation kernel uses Mulberry32, a 32-bit seeded PRNG. Given the same seed, the kernel produces identical agent rosters, lifecycle schedules, and promotion sequences. The only source of variation is the AI commander's decisions. Two runs with the same seed and different personalities produce different outcomes from provably identical starting conditions.

cf. Law & Kelton, Simulation Modeling and Analysis, 5th ed., McGraw-Hill, 2015

Scenario Compiler

Write a scenario JSON draft, or ground it with a prompt, brief, or URL. The compiler calls generateText() to produce TypeScript hooks for progression, event generation, milestones, and political dynamics. A Mars colony, a lunar outpost, an Antarctic station, a corporate org, a submarine crew: same compiler, different world contract, different future. Cost is roughly $0.10 per compile. Results cache to disk.

compileScenario() in paracosm/engine/compiler

Semantic Research Pipeline

Specialist agents search academic literature before making recommendations. The pipeline queries Serper, Tavily, Firecrawl, and Brave in parallel, then applies Cohere Rerank v3.5 for neural relevance scoring. Citations include DOI links. A medical agent recommending radiation shielding cites actual NASA dosimetry guidelines, not a hallucinated source.

Deep research via AgentOS AgentMemory + Cohere Rerank v3.5

Seed Enrichment & Citation Provenance

Pass a brief, paper, or URL into the compiler. An LLM extracts topics, facts, and search queries; AgentOS WebSearchService fans out to Firecrawl, Tavily, Serper, and Brave; results pass through semantic dedup, RRF fusion, and Cohere neural reranking. The resulting KnowledgeBundle ingests into an AgentOS sqlite-backed memory store. Every department analysis recalls semantically and cites the same sources you fed in. The orchestrator guarantees provenance: when the LLM omits citations, the research packet is auto-attached so the report always carries the sources the agent saw.

paracosm compile --seed-text / --seed-url · KnowledgeBundle · recallResearch()

Replay-Verified Determinism

wm.replay(stored) re-executes the deterministic between-turn progression hook from each recorded snapshot, captures fresh snapshots, and compares them under canonical JSON. matches=true proves the kernel is byte-equal-deterministic for this artifact's transitions; matches=false names the first divergence path so you can localize the change. Pillar 2 (Reproducible) is now verifiable in code, not just promised in copy.

Simulation Loop

Anatomy of a Turn

A turn takes 30 to 90 seconds. The engine runs four phases in sequence. Each phase feeds data forward. By the end of a turn, the world state has changed, agents have reacted, and personality traits have shifted.

01

Event Director

Reads world state, prior decisions, and tool intelligence. Generates events that target actual weaknesses and exploit prior choices.

02

Department Analysis

Five department agents analyze the crisis in parallel. Each cites research and can forge new computational tools.

03

Commander Decision

The AI commander weighs department input through the lens of their HEXACO personality profile.

04

Kernel Resolution

Deterministic state machine applies effects. Agents react, form opinions, update relationships. HEXACO traits drift.

Divergence

Same seed. Different commander. Different civilization.

The kernel guarantees identical starting conditions. Seed 950 produces the same 100 agents with the same profiles, the same resource levels, and the same initial events. The only variable is the commander's personality. By turn 3, the two worlds are measurably different.

Digital Twin Use Case

Digital twins, named in the import path

The same engine that runs civilization simulations also models a single subject under counterfactual interventions. Import the curated subpath, supply a SubjectConfig and an InterventionConfig, and the resulting RunArtifact carries both for traceability.

paracosm/digital-twin
import { DigitalTwin, type SubjectConfig, type InterventionConfig } from 'paracosm/digital-twin'; const twin = await DigitalTwin.fromJson(scenarioJson); const subject: SubjectConfig = { id: 'company', kind: 'organization', attributes: { headcount: 100 } }; const intervention: InterventionConfig = { id: 'rif', kind: 'policy', description: '25% RIF', parameters: { percent: 25 } }; const artifact = await twin.simulateIntervention(subject, intervention, leader, { maxTurns: 4 }); console.log(artifact.subject, artifact.intervention);
Scenarios

Two ScenarioPackages ship with the engine

A ScenarioPackage is a JSON file plus runtime hooks. It defines departments, metrics, crisis templates, progression rules, and milestone conditions. The engine reads the package, populates the world, and runs the simulation. No engine code changes required between scenarios.

Flagship

Architecture

Three layers, strict boundaries

The engine layer owns types and the deterministic kernel (71 source files). The runtime layer orchestrates simulations via 8 hook interfaces (18 files). The CLI layer serves the dashboard, batch runner, and HTTP server (45 files). No circular dependencies. The engine never imports from runtime.

FAQ

Frequently asked questions

What is Paracosm?
An open-source TypeScript structured world model for AI agents, available on npm (npm install paracosm). You define or ground a world as a ScenarioPackage: departments, metrics, crisis templates, progression hooks. The engine populates the world, generates events from live state, and resolves outcomes through a seeded deterministic kernel. LLM-driven stages diverge because every prompt carries the leader's HEXACO profile. Two scenarios ship with the package (Mars Genesis, Lunar Outpost). You can compile new ones from a JSON draft with optional prompt, brief, or URL grounding. Paracosm fits the "structured / LLM-based / counterfactual" branch of the world-model taxonomy (Xing 2025, ACM CSUR 2025, Kirfel 2025).
Positioning map: docs/positioning/world-model-mapping.md
How does tool forging work?
When a department agent needs a calculation the codebase doesn't provide, it writes one. A medical officer facing a radiation crisis writes a dose calculator. An engineer writes a structural load model. The code executes in a hardened node:vm sandbox with a wall-clock timeout and observed heap usage. An LLM judge reviews the output for safety and correctness before it enters the decision pipeline. The tool persists for future turns. None of these tools exist in the source code. The agents invent them at runtime.
Implementation: AgentOS EmergentCapabilityEngine + EmergentJudge + CodeSandbox
What is the HEXACO personality model?
HEXACO is a six-factor personality framework from personality psychology: Honesty-Humility, Emotionality, Extraversion, Agreeableness, Conscientiousness, and Openness to Experience. Each agent in Paracosm carries six continuous values between 0 and 1. These values shift across turns through three drift mechanisms: leader influence (agents' traits pull toward their commander's profile), role activation (a chief engineer's conscientiousness trends upward), and outcome reinforcement (surviving a crisis through bold action increases openness).
Ashton, M.C. & Lee, K. (2007). "Empirical, Theoretical, and Practical Advantages of the HEXACO Model of Personality Structure." Personality and Social Psychology Review, 11(2), 150-166. doi:10.1177/1088868306294907
Is the simulation deterministic?
The kernel is fully deterministic. It uses Mulberry32, a 32-bit seeded PRNG. Given seed 950, the kernel produces identical agent rosters, lifecycle schedules, promotion sequences, and resource starting values on every run. The divergence between two parallel worlds comes entirely from the AI commander's decisions. Fix the seed, vary the personality, measure the outcome. This is the counterfactual world simulation model (CWSM) pattern.
PRNG methodology follows Law, A.M. & Kelton, W.D. Simulation Modeling and Analysis, 5th ed., McGraw-Hill, 2015. CWSM framing per Kirfel et al, 2025.
What LLM providers work with Paracosm?
OpenAI (GPT-5.4) and Anthropic (Claude Sonnet 4.6). All roles (commander, departments, crisis director, tool judge) default to the full model for reliable structured output and tool forging. You can override per-role in the setup panel or via the API. Mini models (GPT-5.4-mini, Claude Haiku) are available for cost optimization but produce lower quality results.
Can I build my own scenario?
Two paths. If you want full control, write a ScenarioPackage in TypeScript with custom hooks for progression, crisis generation, milestones, and politics. If you want zero-code authoring today, write a scenario JSON draft and optionally add seed text or a seed URL for grounding, then run it through the Scenario Compiler. A prompt-only wrapper should generate the same JSON contract before simulation. Compilation costs about $0.10 in API calls and caches results to disk. An Antarctic station, a submarine crew, a corporate org, a generation ship: different world contract, same engine, different future.
What verticals does this apply to?
Defense and intelligence use it for wargaming and scenario planning. Corporate strategy teams model acquisition outcomes and leadership dynamics. Game studios generate procedural NPC civilizations and emergent narratives. Academic researchers run controlled experiments in multi-agent decision-making. Government agencies simulate policy impacts before implementation. The common thread: any domain where testing decisions before making them reduces risk.
