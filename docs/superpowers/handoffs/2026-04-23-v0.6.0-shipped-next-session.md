## 🟢 Project summary (2 paragraphs)

Historical note: this handoff was written before the 2026-04-23/24 positioning pass. Current public positioning is **structured counterfactual world model for AI agents**. Prompts, briefs, URLs, or scenario JSON drafts ground a typed `ScenarioPackage`; AI leaders with HEXACO personality profiles run it through LLM-generated crises + department analyses + commander decisions + deterministic kernel progression; and Paracosm produces measurably different outcomes from identical seeds. Built on `@framers/agentos`. Lives at [framersai/paracosm](https://github.com/framersai/paracosm) on GitHub, published as `paracosm` on npm. Hosted demo: `paracosm.agentos.sh`.

The current public contract (post-v0.6.0) is a **universal JSON schema** at `paracosm/schema` that covers three simulation modes: `turn-loop` (civilization sims), `batch-trajectory` (digital-twin like digital-twin), and `batch-point` (one-shot forecasts). Inputs include `SubjectConfig` + `InterventionConfig` primitives. Outputs are a Zod-validated `RunArtifact`. Non-TS consumers get JSON Schema via `npm run export:json-schema`.

- `RunOptions` gained optional `subject?: SubjectConfig` + `intervention?: InterventionConfig` (additive, pass-through).
